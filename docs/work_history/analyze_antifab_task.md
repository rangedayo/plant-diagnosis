⚠ **이번 변수는 "날조 억제" 하나뿐.** "정상 변이 비보고"(하부 노화·말단 갈변을 증상에서 빼기)는 **다음 라운드** — 이번엔 절대 건드리지 말 것. 효과를 분리 측정하기 위함.

## PART A — analyze 프롬프트 수정 (한 변수)

### 추가할 것 = 일반 원칙 한 가지 (증상 종류를 나열하지 말 것)

핵심은 **"관찰 충실성"** 원칙이다. 특정 증상 이름(병반·반점·괴사 등)을 막는 게 아니라, **종류와 무관하게 "실제로 안 보이는 것은 보고하지 않는다"** 는 단일 원칙을 넣는다:

- "이미지에서 실제로 관찰되는 것만 보고하라. 관찰되지 않는 이상 징후를 추론하거나 지어내지 마라." (증상 종류 불문)
- "증상이 없으면 빈 목록도 정상이다. 억지로 증상을 찾아내지 마라."

⚠ **증상 종류를 나열하지 마라.** "병반/반점/괴사를 만들지 마라" 처럼 특정 유형을 적으면, 새 종·새 증상이 들어올 때마다 목록을 늘려야 하는 surface 패치가 된다. 그게 아니라 **"안 보이면 안 적는다"는 추상 원칙 하나**로 모든 종류를 커버해야 한다. (예시가 필요하면 "(예: 반점·변색 등 어떤 이상이든)" 정도로 '종류 불문'을 강조하는 선에서만.)

⚠ **FN 0 사수 — 관찰 충실성은 유지**:
- "실제로 관찰되는 증상은 경미하더라도 모두 보고하라." (날조만 억제하고, 진짜 관찰된 건 빼지 않음 — 이게 '정상 변이 비보고'와의 경계선)

⚠ **surface 패치 금지**: 종 이름·케이스 ID·특정 증상 유형 목록을 박지 말 것. 일반 원칙만.
⚠ **최소 변경**: 날조 억제 원칙만. analyze의 다른 부분·generate·가드·RAG 무변경.
