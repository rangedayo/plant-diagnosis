**자동 매핑 자리** (Claude Code 작업):

각 케이스의 `acceptable_search_hints`를 5 JSON 본문에서 fuzzy 검색해 실제 `card_id` 리스트로 변환:

1. 케이스마다 각 hint(source + keywords)에 대해:
   - 해당 `source`의 JSON 카드들을 순회
   - `title` + `section` + `body`(case-insensitive)에 keywords 중 하나라도 등장하면 매칭 후보
   - 매칭 우선순위: `title` 매칭 > `section` 매칭 > `body` 매칭
2. 케이스별 매칭된 `card_id` 리스트 박음 (중복 제거, 최대 6개 권장)
3. 최종 JSON 스키마:

```json
{
  "case_id": "ext_001",
  "symptom_ko": "...",
  "symptom_en": "...",
  "primary_problem_type": "abiotic",
  "acceptable_card_ids": ["mobot_indoor_xxx", "psu_ucanr_yyy", ...],
  "mapping_notes": [
    {"card_id": "mobot_indoor_xxx", "title": "Overwatering", "matched_by": "title:overwatering"},
    ...
  ]
}
```

4. 매칭 못 찾은 경우 `acceptable_card_ids: []` + `mapping_notes`에 사유 박음 + §5 보고에 명시
5. `acceptable_search_hints` 필드는 최종 JSON에서 제거 (입력 전용)

### 3.2 `scripts/eval_retrieval.py` 신규

기능:
- `eval/golden_set.json` 읽음
- 각 외부 케이스마다 `symptom_en`을 query로:
  - `b_dataset_rag` top 7 검색 (graph.py의 `B_DATASET_TOP_K=7` import 또는 명시)
  - `main_rag` top 3 검색 (`MAIN_TOP_K=3`)
  - 합쳐서 top 10 박음 (graph.py의 `_merge_rag_triples` 로직과 정합 — 가중 정렬 동일하게)
- 케이스별로 메트릭 계산:
  - **Hit@10**: top 10 안에 `acceptable_card_ids` 중 1개 이상 있으면 1, 없으면 0
  - **RR (Reciprocal Rank)**: `acceptable_card_ids` 중 top 10에서 가장 높은 순위의 1/rank (없으면 0)
- 전체 메트릭:
  - **Hit Rate@10** = sum(Hit@10) / N (N = 매핑 성공 케이스 수)
  - **MRR** = mean(RR)
- 결과 박음: `eval/after_phase_b3_retrieval.json`

스키마:
```json
{
  "measured_at": "...",
  "metrics": {
    "hit_rate_at_10": 0.X,
    "mrr": 0.X,
    "cases_evaluated": N,
    "cases_excluded": M
  },
  "per_case": [
    {
      "case_id": "ext_001",
      "symptom_en": "...",
      "primary_problem_type": "abiotic",
      "acceptable_card_ids": [...],
      "top_10": [
        {"rank": 1, "card_id": "...", "source": "...", "problem_type": "...", "sim": 0.X, "title": "..."},
        ...
      ],
      "hit_at_10": 1,
      "reciprocal_rank": 0.5,
      "first_acceptable_rank": 2,
      "excluded_from_metrics": false
    },
    ...
  ]
}
```

매핑 실패 케이스(`acceptable_card_ids` 빈 리스트)는 메트릭 계산에서 제외 + per_case에 `excluded_from_metrics: true` 박음 + 전체 메트릭 계산 시 분모도 줄임 (예: 10 → 9).
