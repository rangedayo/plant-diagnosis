# 작업: 최근 진단 기록 카드 — 개수 2개 + 카드 디자인 개편 (D 후속)

> 항목 D(홈 최근 진단 기록 fan-out) 구현됨, 로컬 미푸시. **표시 개수·카드 레이아웃만 조정.** 조회 로직·status 라벨·네비는 무변경.
> 워크플로우: CLAUDE.md §5. recon-first, 변수 격리, 푸시 보류→검토. PowerShell.

## 1. 이번 변수
홈 "최근 진단 기록"의 **표시 개수(5→2)와 카드 디자인**만 변경. fan-out 조회·정렬·history 네비·status 색 로직은 동결.

## 2. 변경 내용

### A. 개수
- 표시 최신 진단 **2개**로 제한(현재 더 많이 노출). 정렬은 기존(최신순) 유지.

### B. 카드 디자인 (목표 레이아웃 묘사)
현재 = 작은 썸네일 + 한 줄 행. **목표 = 사진 큰 카드.** 각 카드 구성:
- **왼쪽**: 큰 식물 사진(세로로 꽉 찬 라운드 이미지, 카드 높이만큼).
- **오른쪽**: 위→아래로 ① 식물명(굵게, 크게) ② status 배지(사용자 라벨 — 건강/양호 (가벼운 주의)/주의 관찰, 기존 status 색 pill) ③ 날짜 ④ 요약 한 줄(`structured_result.summary`/`record.summary`, 1줄 말줄임) ⑤ 우하단 화살표(상세 진입).
- 카드: 흰/연그린 배경, 큰 라운드(기존 `radius-card` 계열), 기존 카드 그림자·토큰 재사용.

### C. 헤더
- 섹션 헤더 "최근 진단 기록"(시계 아이콘) 오른쪽에 **"전체 보기 >" 링크** 추가 → "내 식물"(myPlants) 화면으로 이동(기존 탭/네비 재사용).

### D. 배치
- 모바일 폭(~380px) 기준 **풀폭 카드를 세로로 2개 스택**(목표 이미지의 가로 카드를 풀폭화). 가로 2-up은 폭이 좁아 금지.
- 날짜 표기는 기존 상대 표기("오늘"·"5일 전") 유지(목표 이미지의 절대일자로 바꾸지 않아도 됨 — 더 친근).

## 3. 보존 (변경 금지)
- fan-out 조회·정렬·N 산출 로직(개수 상한만 2로), status→라벨/색 매핑, 기록 카드 탭→history 네비.
- 로그인/미로그인 분기·더미 금지 원칙 유지. **진단 1개·0개일 때 처리**(1개면 1장, 0개면 기존 empty state).
- 다른 화면·챗봇·진단 파이프라인 무접촉.

## 4. 합성 검증
- `tsc --noEmit`·`next build` 통과.
- 데이터 2개 초과 시 2장만, 정확히 2개, 1개, 0개(empty) 각 분기 정상.
- "전체 보기" → myPlants 이동, 카드 탭 → 상세 네비 정상.
