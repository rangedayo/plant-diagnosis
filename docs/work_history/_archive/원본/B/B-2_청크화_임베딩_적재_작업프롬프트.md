## 1. 컨텍스트 (한 줄)

[B-1] b_dataset raw 수집 푸시 후 → **[B-2] 청크화·임베딩·적재**. NCPMS 컬렉션 폐기 + `b_dataset_rag` 신규 컬렉션 (82 카드) + `main_rag` 유지 (21 카드). 검색 비중 재조정 (`MAIN_TOP_K=3`, `B_DATASET_TOP_K=7`). 단일 atomic 커밋.

핵심 성격:
- **NCPMS 자리에 b_dataset 들어감** — v14 가설 "b_dataset이 1단계 FP 본질 직격" 검증 자리
- **단일 atomic 커밋** — 새 컬렉션 적재·검증 → NCPMS 폐기 → 검색 코드 갱신 → quick 측정 순서. 부분 적재 금지
- **변수 격리** — main_rag·임베딩 모델·UC IPM 페널티 등 손대지 않음. 본 단계는 b_dataset 적재 + NCPMS 폐기 + 비중 조정만
- **회귀 게이트 박지 않음** — quick 1회 측정은 적재 망가짐 검출 용도. F 옵션 통과 판정은 [B-3]에서

원칙:
- **2계층 sanity check 통과만이 본 단계 게이트** — 빈 청크 0, 임베딩 nearest neighbor 5 쿼리 사람 검토 통과
- **카드=청크 1:1** — `build_main_rag.py` 패턴 일관성
- **`ncpms` 식별자 모두 `b_dataset`으로** — 상수·logger·메타 값 일관성 갱신
- **main_rag 이름 변경 금지** — 별도 단계 자리. 본 작업에서 손대지 않음

## 3. 작업 항목

### 3.1 `scripts/build_b_dataset_rag.py` 작성·실행 (주 작업)

5 JSON → 청크 변환 + 임베딩 + Chroma 적재.

**입력**: `data/raw/b_dataset/*/*.json` 5자료
**출력**: ChromaDB `b_dataset_rag` 컬렉션 (`data/vector_db/`)

#### A. 청크 처리 (영역 1 A — 카드=청크 1:1)

- 각 JSON 로드 → `cards` 배열 순회 → 1 카드 = 1 청크
- 청크 본문 = `title + ": " + body` (제목·본문 결합 — `build_main_rag.py`의 `flatten_document` 패턴 일관성)
- 청크 ID = 카드의 `id` 필드 그대로 (`psu_ucanr_001`, `mobot_indoor_005` 등)

#### B. 메타데이터 6 필드 (영역 3 B)

```python
{
    "source": <"psu_ucanr"|"psu_indoor"|"mu_trinklein"|"mobot_indoor"|"mobot_herb">,
    "source_id": <카드 JSON의 source 필드값>,
    "section": <카드의 section>,
    "title": <카드의 title>,
    "problem_type": <자동 분류, 아래 매핑 룰>,
    "card_id": <카드의 id>,
}
```

Chroma 메타데이터는 str 값만 허용 — null/None은 빈 문자열 `""` 또는 `"generic"`으로.

#### C. `problem_type` 자동 분류 매핑 룰

| 자료 (`source`) | `section` 매칭 패턴 | `problem_type` |
|---|---|---|
| psu_ucanr | "Pest" 포함 | `pest` |
| psu_ucanr | "Disease" 포함 | `disease` |
| psu_ucanr | "Abiotic" 포함 | `abiotic` |
| psu_indoor | "Pest" 포함 | `pest` |
| psu_indoor | "Disease" 포함 | `disease` |
| mu_trinklein | 첫 2 카드 (id `mu_trinklein_001`·`002`) | `frame` |
| mu_trinklein | 그 외 | `general` |
| mobot_indoor | "Environmental Conditions" | `env` |
| mobot_indoor | "Insects" | `pest` |
| mobot_indoor | "Diseases" | `disease` |
| mobot_indoor | "Nutrients" | `nutrient` |
| mobot_herb | "Insects" 또는 "insect" 포함 | `pest` |
| mobot_herb | "Disease" 포함 | `disease` |
| mobot_herb | 그 외 | `general` |

매칭 실패 카드는 `general` 디폴트. MU Trinklein 첫 2 카드(frame 정의)는 id로 직접 박기 — section 파싱 의존 안 함.

#### D. 임베딩 (영역 5 A)

```python
embeddings = OpenAIEmbeddings(openai_api_key=openai_key)  # model 인자 없음 — main_rag와 동일
vectors = embeddings.embed_documents(documents)
```

#### E. Chroma 적재 (영역 4 A)

`scripts/build_main_rag.py`의 `persist_main_rag` 패턴 그대로:

```python
client = chromadb.PersistentClient(path=str(vector_db))
try:
    client.delete_collection("b_dataset_rag")  # 멱등성
except Exception:
    pass
collection = client.create_collection(
    name="b_dataset_rag",
    metadata={"hnsw:space": "cosine"},
)
ids = [card_id for card_id in card_ids]  # 카드 id 그대로
collection.add(ids=ids, embeddings=vectors, documents=documents, metadatas=metas)
```

### 3.2 `app/graph.py` 검색 코드 갱신

여러 자리 동시 변경. [1-8] retrieve 정비 패턴 참조.

#### A. 상수 변경

```python
# Before
MAIN_TOP_K = 7
NCPMS_TOP_K = 3
NCPMS_SIM_WEIGHT = 0.8

# After (영역 6 C)
MAIN_TOP_K = 3              # 비중 재조정: main 보조
B_DATASET_TOP_K = 7         # 비중 재조정: b_dataset 메인
B_DATASET_SIM_WEIGHT = 1.0  # 페널티 제거 (가설 직격). 상수 유지로 명명 일관성 + 향후 조정 여지
```

#### B. 검색 호출 자리 (`retrieve_node` 내부)

- `_chroma_query_sync(query, db_path, NCPMS_TOP_K, "ncpms_rag")` → `_chroma_query_sync(query, db_path, B_DATASET_TOP_K, "b_dataset_rag")`
- 결과 변수명 `docs_ncpms, metas_ncpms, sims_ncpms, err_ncpms` → `docs_b, metas_b, sims_b, err_b`

#### C. `_merge_rag_triples` 함수

- 인자명 `ncpms_triples` → `b_dataset_triples`
- 인자명 `ncpms_top_k` → `b_dataset_top_k`
- 내부 변수 `n_enriched` → `b_enriched`, `ncpms_selected` → `b_selected` 등 일관성
- logger 메시지 `"retrieve: merge_rag ncpms_top_k=%d main_top_k=%d ..."` → `b_dataset_top_k`로

#### D. `_merge_similarity_for_ranking` 함수

```python
# Before
if rag_source == "ncpms":
    s *= NCPMS_SIM_WEIGHT
    logger.debug("rank: ncpms sim weight raw=%.4f -> %.4f", raw_cosine, s)

# After (분기 유지 — 로그 명시성)
if rag_source == "b_dataset":
    s *= B_DATASET_SIM_WEIGHT
    logger.debug("rank: b_dataset sim weight raw=%.4f -> %.4f", raw_cosine, s)
```

`B_DATASET_SIM_WEIGHT=1.0`이라 사실상 no-op이지만 분기·로그는 유지 — [B-3] 측정 후 가중치 조정 자리 박아둠.

`main_rag`의 `UC_IPM_SOLUTION_SIM_PENALTY=0.85` 분기는 **그대로** (영역 6 변수 격리).

#### E. `_doc_plant_for_ranking` 함수

```python
# Before
if rag_source == "ncpms":
    return _doc_first_token_as_crop(doc)

# After — 분기 제거
# (b_dataset은 plant_name 메타 없음, 기존 main_rag 분기만 살아있게)
```

`_doc_first_token_as_crop` 헬퍼는 사용처 0이 되면 제거. grep으로 확인:
```bash
grep -rn "_doc_first_token_as_crop" app/ tests/
```
다른 사용처 발견되면 보고.

#### F. `_tag_triples_rag_source` 호출 자리

```python
# Before
t_ncpms = _tag_triples_rag_source(_triples_from_chroma(...), "ncpms")

# After
t_b = _tag_triples_rag_source(_triples_from_chroma(docs_b, metas_b, sims_b), "b_dataset")
```

#### G. 로그·DEBUG 출력

```python
# Before
print("[DEBUG] ncpms_docs:", len(docs_ncpms))
logger.warning("retrieve: ncpms_rag 시스템 오류: %s", err_ncpms)

# After
print("[DEBUG] b_docs:", len(docs_b))

```

`if err_ncpms and err_main:` → `if err_b and err_main:`. 두 컬렉션 모두 실패해야 `rag_failed=True`로 박는 패턴 그대로.

- **raw JSON body 변형 금지**: [B-1]에서 정제된 body를 그대로 청크 본문으로 사용. 추가 trim·공백·인코딩 변환 X.
- **카드 ID 충돌 방지**: 청크 ID = raw JSON의 `id` 그대로 (`psu_ucanr_001`, `mobot_indoor_005` 등). source prefix가 자료 간 충돌 방지.
- **`title + ": " + body`** 청크 본문 결합 패턴: title이 검색 신호로 작용. `build_main_rag.py`의 `flatten_document` 일관성.
- **임베딩 모델 명시 변경 금지**: `OpenAIEmbeddings()` 기본값 그대로 (main_rag와 동일). 명시 모델 변경은 별도 단계.
