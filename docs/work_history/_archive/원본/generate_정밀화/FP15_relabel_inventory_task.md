# [라벨 전수 재검 1단계] FP 15건 인벤토리 + 판정 워크시트 (read-only) — 작업 프롬프트

## 0. 맥락 + 목적

R12d-1 측정에서 healthy→비건강 FP가 15건. 이 중 일부는 **모델 잘못이 아니라 GT 라벨 잘못**일 가능성이 확인됨:
- self_dracaena_003·004·006: 사용자 본인 식물. 직접 관찰 결과 "안정적으로 새 잎 내며 생장 중, 하부 잎 자연 노화" → **건강이 맞음** (모델의 건조 판정이 라벨 대비 오답이지만, 실제로는 hard case).
- inat_spathiphyllum_002: 아래쪽 잎 광범위 시듦 → **비건강이 맞음** (현 GT=건강이 오류).

→ FP 15건 전수를 같은 기준으로 재검해서 "라벨 잘못(GT 정정/제외)" vs "진짜 모델 잘못"을 분리한다. 이미지 판정은 사용자가 웹에서 수행하므로, **이번 작업은 그 판정을 위한 인벤토리·워크시트·이미지 경로 정리**까지.

⚠ **read-only. GT 라벨·코드·평가셋 일절 변경 금지.** 기존 `eval/after_acc_r12d1_remove_surface.json` 분석 + 이미지 경로 확인만.

- observed_symptoms (전체)
- pred_status
- top_3 problem_type
- 원본 이미지 파일 경로

## PART B — 재검 rubric (워크시트 헤더에 명시)

합의된 판정 기준:

| 조건 | 판정 |
|---|---|
| `self_` + 전체 생장 안정 + 갈변이 말단(잎끝)·하부 국한 | **건강 유지** (hard case로 인정) |
| `self_` + 광범위/진행성 손상 (전체 시듦·줄기 손상) | **비건강 정정** |
| `inat_` + 광범위 시듦/명백한 이상 | **비건강 정정** |
| `inat_` + 경미·애매 | **ambiguous 제외** |
| 이진(건강/비건강) 판정이 본질적으로 모호 | **ambiguous 제외** |

⚠ 비건강으로 정정하더라도 **원인 status(건조/과습/병해/영양)는 별개 판단** — 이미지로 원인이 불명확하면 "비건강·원인미상"으로 표기하거나 ambiguous. 모델이 "건조"로 찍었다고 GT를 자동으로 "건조"로 정정하지 말 것.

## PART C — 판정 워크시트 생성

`docs/work_history/FP15_relabel_worksheet.md`에 표로:

| case_id | 종 | 출처 | observed_symptoms | 현 GT | pred_status | top_3 | 이미지 경로 | 재검 판정(빈칸) | 재검 status(빈칸) | 근거(빈칸) |

- 마지막 3칸(재검 판정/status/근거)은 **빈칸** — 사용자가 이미지 보고 채울 자리.

  - dracaena_003·004·006 → 건강 유지 (사용자 실측)
  - spathiphyllum_002 → 비건강 정정 (광범위 시듦, 원인 미상)
