## 배경

R4 핫픽스 푸시 후, **실제 백엔드로 e2e를 처음 끝까지 관통**하면서 발견된 UI 정합 4건. 전부 프론트 표시 문제이고 데이터·동작 로직과 무관.

※ **케어 화면은 이번 e2e에서 진입·렌더 확인됨**(몬스테라 = `care_guide` 매칭). 케어 화면의 동작·레이아웃 자체는 정상이라 **케어 전용 수정은 없음**. 다만 케어 화면 상단에도 옛 브랜드명 "Plant Butler"가 노출되므로, 그건 **작업 1(브랜딩 전수 교체)에서 함께 처리**된다.

## 작업 1 — 브랜딩: "Plant Butler" → "Plantia"

### 증상
로딩 화면 상단 등에 옛 이름 **"Plant Butler"** 가 노출됨. 정식 브랜드명은 **"Plantia"**.
**(e2e 캡처 확인)** 로딩뿐 아니라 **결과 화면·케어 가이드 화면 최상단에도 동일하게 "Plant Butler"가 노출됨** → 같은 출처(전역 헤더로 추정, 작업 2에서 확정)일 가능성이 높음.

### read-only 게이트
- 리포 내 "Plant Butler" 문자열이 **어디어디** 나오는지 전수 확인(로딩 화면, 전역 헤더, 결과·케어 화면 상단, `<title>`, 메타 등).

### 수정
- 노출되는 "Plant Butler"를 **"Plantia"** 로 교체(전역 헤더가 출처라면 한 곳 수정으로 로딩·결과·케어가 함께 바뀜).
- (작업 2와 겹치는 부분 주의 — 결과 화면의 **네비 헤더 복원**은 작업 2에서 별도 처리. 작업 1은 "이름 문자열 교체"만.)

## 작업 2 — 결과 화면 헤더: "진단 결과" 네비 헤더 복원 (★ 조사 — 캡처로 정황 확보)

### 증상
결과 화면 상단에 **"Plant Butler"** 가 뜨고, 정본(`plantia_combined.html`)의 **`< 진단 결과 ✦` 네비 헤더**(뒤로 화살표 + 가운데 제목 + 우측 스파클)가 안 뜸.

### 캡처로 드러난 구조 정황 (read-only로 반드시 확정할 것 — 단정 금지)
- **결과 화면과 케어 화면 둘 다 최상단에 동일한 "Plant Butler"** 가 떠 있음 → "Plant Butler"는 특정 뷰 소속이 아니라 **전역 래퍼 헤더**(모든 screen 위에 얹히는 구조)일 가능성이 큼.
- 그런데 **CareGuideView는 자기 네비 헤더 `< 케어 가이드`(뒤로 + 제목)를 정상적으로 렌더**하고 있음. 반면 **ResultView에는 `< 진단 결과 ✦` 네비 헤더가 없음**.
- 즉 **두 뷰의 헤더 처리 방식이 다름**(케어=뷰 자체 네비 헤더 있음 / 결과=없음). → 결과 쪽이 케어와 동일 패턴을 따르도록 맞추는 게 일관성 측면에서 자연스러운 방향.

### read-only 게이트 (보강)
- 위 정황 확정: "Plant Butler" 헤더가 **어디서 렌더되는지** 특정(전역 래퍼 `pages/index.tsx`/`_app.tsx`에서 모든 screen에 얹히는지).
- **CareGuideView의 `< 케어 가이드` 헤더 블록 마크업·스타일**이 어떻게 구성돼 있는지 파악.
- `ResultView.tsx`에 `plantia_combined.html`의 `< 진단 결과 ✦` 헤더 블록이 **존재하는지 / 왜 안 뜨는지**, CareGuideView 대비 무엇이 빠졌는지 대조.
- 구조 파악 결과 보고(전역 헤더가 결과 화면까지 덮는 형태인지, 케어/결과 헤더 처리 차이가 무엇인지).

### 수정
- 결과 화면이 정본대로 **`< 진단 결과 ✦` 헤더**를 갖도록 — 가능하면 **CareGuideView가 이미 쓰는 네비 헤더 패턴을 그대로 따라** 일관성 확보.
  - 뒤로(`<`)는 **기존 뒤로가기 동작(result→home)과 연결** — 새로 만들지 말고 현재 전환 로직 재사용. 누락 금지.
  - 스파클(✦)은 정본 기준 장식(액션 없음)으로 렌더.
- 전역 "Plant Butler" 헤더가 결과 화면을 덮고 있으면, **결과 화면에선 그 헤더가 안 나오도록** 처리(로딩 등 다른 화면의 헤더는 작업 1 결과대로 "Plantia" 유지). 케어 화면이 이미 어떻게 처리하는지(전역 헤더 + 자기 네비 공존인지, 대체인지) 보고 동일 규칙 적용.
- 결과 화면에 **업로드한 실제 사진이 뜨는 것은 정상**(시안 일러스트는 목업 자리표시, 실제는 `imageUrl`). 사진 표시는 건드리지 말 것 — 헤더만.

## 작업 3 — 처방 리스트: 아이콘·텍스트 정렬

### 증상
"처방"(action_plan) 항목의 **체크 아이콘과 텍스트가 한 행에서 세로 가운데로 정렬되지 않고 어긋나** 보임.

### read-only 게이트
- `ResultView.tsx`의 처방 리스트 항목 마크업·`<style jsx>`를 `plantia_combined.html`의 해당 리스트와 대조.

### 수정
- 정본대로 아이콘·텍스트가 한 행에서 세로 가운데 정렬되도록 보정(flex `align-items` 등). 항목 다수 줄 텍스트에서도 정렬 유지.

## 작업 4 — 홈: 카메라 FAB ↔ "사진 촬영" 간격 축소

### 증상
홈 화면에서 **카메라 FAB와 "사진 촬영" 라벨 사이 세로 간격이 정본보다 큼**(현재 렌더가 더 벌어져 있음).

### read-only 게이트
- `HomeView.tsx`의 FAB~"사진 촬영" 라벨 간 마진/간격을 `plantia_home.html`과 대조.

### 수정
- 정본 기준으로 간격 축소(FAB가 "사진 촬영"에 더 가깝게). 정본이 곧 목표값.
