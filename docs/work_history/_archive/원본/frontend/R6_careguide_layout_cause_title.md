# 케어 가이드 레이아웃 보정 + 결과 설명 카드 제목 (3건 + 시안 갱신)

## 배경

R5 푸시 후 **prod e2e(몬스테라)로 실데이터 화면을 보며** 발견된 추가 보정 3건. 전부 프론트 표시·문구·레이아웃 문제이고 데이터·동작 로직과 무관.

- **(작업 1)** 결과 화면 "원인 설명" 제목이 건강한 식물엔 "원인"이라 어색 → 상태 무관 제목으로 교체.
- **(작업 2)** 케어 화면 하단 "비료" 라벨이 세로로 쪼개져 렌더됨(값 텍스트가 길어 라벨 칸이 짜부됨).
- **(작업 3)** 케어 화면 상단 4개 항목(토양·광량·배치·관리 난이도)이 가로 4열이라, 실데이터 텍스트(특히 "배치")가 길어 좁은 열에서 줄바꿈이 지저분하고 높이가 들쭉날쭉 → 세로 1열 스택으로.

※ **작업 3은 정본 케어 시안(현재 4열)에서 의도적으로 벗어나는 디자인 결정**입니다(실데이터 텍스트 길이 기준, 사용자 확정). 코드 변경과 함께 **정본 케어 시안 문서도 새 세로 행 레이아웃으로 갱신**해 "시안=권위" 원칙을 유지합니다.

## 공통 규칙 (유지)

- 변경 전 **read-only로 현재 상태 파악**, 정본과 불일치 시 보고하고 멈춤.
- `styled-jsx` scoped 유지, `@import` 금지, 전역 오염 금지.
- 백엔드(`app/*`)·`eval/*`·`baseline.json`·`data/*` **무접촉**. 프론트(`components/*`, `pages/*`) + `docs/design/*`(작업 3 시안 갱신)만.
- **동작·전환 로직 무변경**(화면 전환, 뒤로가기, `care_guide` 조건부 렌더, 결과 실사진 표시 등). 이번 건 전부 표시·문구·레이아웃 보정.
- 검증용 임시 파일·스크린샷은 커밋 제외(삭제).
- 커밋 분리: `docs:`(이 프롬프트) / `docs:`(시안 갱신) / `fix:`(코드). 의미별 atomic.

## 작업 1 — 결과 화면: "원인 설명" → "이렇게 판단했어요"

### 증상
결과 화면 설명 카드 제목이 **"원인 설명"** 인데, 건강한 식물에도 "원인"이라 어색하게 읽힘.

### read-only 게이트
- `ResultView.tsx`에서 "원인 설명" 제목 문자열 위치 확인(다른 곳에도 동일 문자열이 쓰이는지).

### 수정
- 제목 문구를 **"원인 설명" → "이렇게 판단했어요"** 로 교체.
- 카드 본문(`cause`)·아이콘·구조는 **무변경**, 제목 텍스트만 교체.

## 작업 2 — 케어 화면: "비료" 라벨 한 줄로

### 증상
케어 하단 스펙 행(**생육 적온 · 습도 · 비료**) 중 "비료" 라벨이 **"비/료" 2줄로 세로 쪼개짐**. 값 텍스트("비료를 보통 요구함(봄, 여름에 수용성 비료를 2주에 한 번 줌)")가 길어 라벨 칸이 짜부된 것. 생육 적온·습도는 값이 짧아 정상.

### read-only 게이트
- `CareGuideView.tsx`의 해당 스펙 행 마크업·`<style jsx>` 확인(라벨/값 flex 구조, 현재 라벨에 `white-space`/`flex-shrink`/`min-width` 유무).

### 수정
- 라벨이 **항상 한 줄**로 렌더되도록: 라벨에 `white-space: nowrap` + `flex-shrink: 0`(필요 시 적절한 `min-width`). 값(설명)이 남은 폭을 차지하고, 길면 **값 쪽에서** 줄바꿈되게.
- 세 행(생육 적온·습도·비료) **모두 동일 규칙** 적용해 일관 유지.

---

## 작업 3 — 케어 상단 4항목: 가로 4열 → 세로 1열 스택 (+ 시안 갱신)

### 증상
케어 상단 "지속 관리법"의 4항목(**토양 · 광량 · 배치 · 관리 난이도**)이 가로 4열. 실데이터 텍스트(특히 배치: "거실 창측(실내깊이 150~300cm),발코니 내측(...),발코니 창측(...)")가 길어 좁은 열에서 줄바꿈이 지저분하고 항목 간 높이가 들쭉날쭉.

### read-only 게이트
- `CareGuideView.tsx`의 4항목 영역 마크업·CSS(현재 4열 그리드/플렉스 구조) 확인.
- **정본 케어 시안이 어느 문서에 있는지 특정**(추정: `docs/design/plantia_combined.html`의 케어 섹션) + 그 4열 레이아웃 블록 위치.
- 하단 스펙 행(작업 2)과 **항목 내부 구조 대조**(아이콘+라벨+설명 패턴을 통일할 수 있는지).

### 수정 (코드)
- 4항목을 **세로 1열 스택**으로(각 항목 = 전체 폭 한 행 block). 4열 그리드 제거.
- 각 항목 내부 배치는 **하단 스펙 행과 통일**하면 일관적(아이콘+라벨 + 설명을 한 행에). 단, 정확한 내부 정렬(아이콘·라벨·설명 위치)은 read-only 대조 후 **더 깔끔한 쪽을 제안·보고**.
- 설명이 길어도 전체 폭 행이라 줄바꿈 여유가 확보되는 게 목적.

### 수정 (시안 정합)
- read-only로 특정한 **정본 케어 시안 문서의 해당 4열 섹션을 새 세로 행 레이아웃으로 갱신** → 시안과 구현 일치 유지. (`docs:` 커밋으로 분리)

## 검증
- `tsc --noEmit` 0.
- (가능 범위) dev 헤드리스 캡처(Tabler 웹폰트 로드 대기 `--virtual-time-budget`)로:
  - 결과 설명 카드 제목 **"이렇게 판단했어요"**,
  - 케어 **"비료" 라벨 한 줄** + 값 정상 줄바꿈,
  - 케어 상단 **4항목 세로 행 스택**(배치 긴 텍스트도 깔끔).
- **동작·전환 보존** 확인(뒤로가기, care 진입 조건, 결과 실사진).
- **정본 케어 시안 문서가 새 레이아웃으로 갱신**됐는지 확인.
- ※ 상태 구동 화면(로딩/케어)의 최종 육안은 사용자 prod e2e(`npm run build && npm run start`, 깔끔히 보려면 `.next` 삭제 후)에서 수행.
