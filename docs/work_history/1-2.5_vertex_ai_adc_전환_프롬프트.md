> 목적: 기존 Google AI Studio API 키 인증을 **Vertex AI ADC 인증**으로 전환. SDK는 `google-genai` 유지하고 **클라이언트 초기화 방식만** 변경. AI Studio 인증은 fallback으로 보존.

> 사전 진단: `docs/work_history/1-2.5_vertex_ai_adc_전환_진단.md` (사용자 확정 완료)

## 확정된 결정 (변경 금지)

1. **SDK 유지**: `google-genai` 그대로. `google-cloud-aiplatform` 절대 도입 금지.
2. **region**: `asia-northeast1` (도쿄). 서울(asia-northeast3)은 Gemini 2.5 Pro 미지원이므로 후보에서 제외.
3. **인증 분기**: `GeminiProvider.__init__` 내부에서 환경변수로 자동 분기 (옵션 A).
   - `GOOGLE_CLOUD_PROJECT` 있으면 Vertex 모드 (`vertexai=True, project, location`)
   - 없고 `GEMINI_API_KEY` 있으면 AI Studio 모드 (`api_key`)
   - 둘 다 없으면 RuntimeError
4. **회귀 게이트** -5%p: 인프라 변경이라 적용. 미달 시 revert.
5. **테스트·문서 동시 갱신**: 단일 커밋.

## 작업 묶음

### 1. `app/vision/gemini.py` — `__init__` 환경 분기

**변경 전 시그니처** (사전 확인 C에서 읽은 그대로):
```python
def __init__(self, *, api_key: str, system_prompt: str = "", ...) -> None:
    self._client = genai.Client(api_key=api_key)
    ...
```

**변경 후 시그니처**:
```python
def __init__(
    self,
    *,
    api_key: str | None = None,
    project: str | None = None,
    location: str | None = None,
    system_prompt: str = "",
    ...
) -> None:
    project = project or os.getenv("GOOGLE_CLOUD_PROJECT")
    location = location or os.getenv("GOOGLE_CLOUD_LOCATION") or "asia-northeast1"
    if project:
        self._client = genai.Client(vertexai=True, project=project, location=location)
        self._auth_mode = "vertex"
        self._project = project
        self._location = location
    elif api_key:
        self._client = genai.Client(api_key=api_key)
        self._auth_mode = "ai_studio"
    else:
        raise RuntimeError(
            "GOOGLE_CLOUD_PROJECT(Vertex 모드) 또는 GEMINI_API_KEY(AI Studio 모드) 중 하나 필요."
        )
    ...
```

- `os` 모듈 import 필요 시 추가.
- 기존 인스턴스 속성·`analyze` 메서드 본체는 **절대 손대지 마라**. SDK API는 두 모드에서 동일.
- `_auth_mode` 속성은 통합 테스트와 보고용. private 속성.

### 2. `app/main.py` — lifespan GeminiProvider 생성 호출부

현재 코드를 사전 확인 C에서 읽은 뒤, 호출부가 다음 두 형태 중 하나이면 적절히 갱신:

**현재 형태 A** (api_key 명시):
```python
vision_provider = GeminiProvider(
    api_key=model_utils.get_gemini_api_key(),
    system_prompt=prompts.ANALYZE_SYSTEM,
)
```

**변경 후 (옵션 A — 환경 분기 자동)**:
```python
# Vertex 모드 우선, GEMINI_API_KEY는 fallback
vision_provider = GeminiProvider(
    api_key=model_utils.get_gemini_api_key(),  # fallback용. Vertex 환경변수 있으면 무시됨
    system_prompt=prompts.ANALYZE_SYSTEM,
)
```

- `get_gemini_api_key()` 헬퍼는 `model_utils.py`에서 유지. AI Studio fallback 분기 위해 살아 있어야 함.
- 호출부 변경 최소 (1줄 안 바뀌어도 됨, 환경변수가 알아서 분기 결정).

### 3. `scripts/run_eval.py` + `scripts/eval_rag.py` — 호출부

위 main.py와 동일 패턴. 별도 변경 거의 없음. 사전 확인 C에서 읽은 호출부 시그니처와 일치하는지만 확인.

### 4. `tests/vision/test_gemini_provider.py` — skipif 확장 + Vertex 통합 테스트

**skipif 조건 확장**:
```python
# 변경 전
@pytest.mark.skipif(not os.getenv("GEMINI_API_KEY"), reason="API key not set")

# 변경 후 — 두 모드 중 하나라도 가능하면 통합 테스트 실행
@pytest.mark.skipif(
    not (os.getenv("GOOGLE_CLOUD_PROJECT") or os.getenv("GEMINI_API_KEY")),
    reason="GOOGLE_CLOUD_PROJECT(Vertex) 또는 GEMINI_API_KEY(AI Studio) 중 하나 필요",
)
```

**Vertex 모드 통합 테스트 1건 추가**:
```python
@pytest.mark.integration
@pytest.mark.skipif(
    not os.getenv("GOOGLE_CLOUD_PROJECT"),
    reason="GOOGLE_CLOUD_PROJECT 미설정 — Vertex 모드 테스트 스킵",
)
async def test_vertex_mode_authenticates_and_analyzes():
    """Vertex 모드로 self_haengun_001.jpg 1회 호출. ADC 자동 인증 검증."""
    provider = GeminiProvider(
        system_prompt="이 이미지를 한국어로 분석해서 6필드 JSON으로 반환하라",
    )
    assert provider._auth_mode == "vertex"
    
    image_bytes = (
        pathlib.Path("test_data/main_eval/images/self_haengun_001.jpg")
    ).read_bytes()
    result = await provider.analyze(
        VisionInput(image_bytes=image_bytes, mime_type="image/jpeg")
    )
    assert result.plant_name  # 6필드 채워졌는지
    assert isinstance(result.observed_symptoms, list)
```

- 기존 단위 테스트(mock 5건)·기존 AI Studio 통합 테스트는 **건드리지 마라**.
- 새 Vertex 테스트는 ADC 인증과 SDK Vertex 모드 동작을 1회 실증.

## 롤백 분기

### 분기 1 — 게이트 미달 (-5%p 초과 회귀)
`git revert HEAD` 즉시. 사용자에게 회귀 케이스별 분석 요청.

### 분기 2 — safety filter 차단
Vertex Gemini가 plant 이미지 일부를 차단할 가능성 (낮음, 그러나 발생 시):
1. `GeminiProvider.__init__` 또는 `analyze` 메서드에 `safety_settings` 인자 추가
2. `BLOCK_NONE` 또는 `BLOCK_ONLY_HIGH` 설정
3. 사용자에게 보고 후 재측정

### 분기 3 — `response_schema=AnalyzeResult` Vertex 모드 미지원
SDK 실증 결과 [1-2]에서 AI Studio 모드는 작동 확인. Vertex 모드도 동일 동작 기대(같은 SDK·같은 모델). 만약 차이 발견:
- TypeError·schema 오류 시 보고
- 임시 우회: `response_mime_type="application/json"`만 사용 + Pydantic 후처리
