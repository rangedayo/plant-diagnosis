## 0. 한 줄 요약

1차 진단 정확도 트랙의 목표는 **이진 FP/FN(건강 vs 비건강)을 튼튼히 측정하고, 표본이 확보된 status(병해)의 분류 정밀도를 혼동표로 측정**하는 것이다. 핵심 병목은 **병해 표본 부족**과 **5-status 정답 라벨 부재**였고, 본 문서는 그 해소 방식과 **측정 가능 범위의 정직한 경계**를 합의한다.

## 1. 배경 — 현황과 갭

### 실측 평가셋 (run_eval이 실제 소비)

- 단일 소스: `test_data/main_eval/labels.json`, **9종 33장**.
- 구성: 정상 28 / 증상 5.
  - self_captured: 행운목(2/5), 드라세나 송오브인디아(5/0)
  - iNaturalist(수동 큐레이션) 7종: 아글라오네마·접란·스킨답서스·고무나무·몬스테라·산세베리아·스파티필름 — 각 3/0 (전부 정상)
- L0 기준선은 이미 이 9종 33장 위에서 측정됨.

### 두 개의 갭

1. **병해(FN) 표본 빈약** — 증상 5장이 전부 행운목 단일종. FP/precision 축은 9종으로 튼튼하나, **FN·recall 측정은 단일종에 묶여 사실상 불능**.
2. **5-status 정답 라벨 부재** — 현 라벨은 `is_healthy`(이진) + `symptoms` + 자유서술 `diagnosis`뿐. status 5종(`건강`·`과습`·`건조`·`병해 의심`·`영양 부족`) 정답이 없어 **status별 혼동을 점수화할 수 없음**.

### 자산 현황

- **PlantVillage 50장** — 병해 라벨 완료(`test_data/plantvillage_50/labels.json`), run_eval 미연결. 작물(토마토·감자·사과) 도메인이라 메인 부적합 → 보조용(§4 결정3).
- **Wikimedia** — 실제 수집 시도 결과 **수율 0**(5장 중 식물 실사진 1장, 그마저 건강·나머지는 일러스트/책 페이지). **폐기**(§4 결정1).
- **Money Plant 데이터셋(신규 후보)** — 스킨답서스(=평가셋 종) 병해, CC BY 4.0, 스마트폰 in-the-wild. 편입 후보(§4 결정1).

## 2. 1차 목표 (이번 트랙의 스코프) — v2 재정의

측정은 **두 층위**로 함께 수행한다:

| 측정 | 무엇 | 1차 달성 |
|---|---|---|
| **이진 FP/FN** | 건강 vs 비건강 (FN=아픈 걸 놓침, FP=건강을 아프다 함) | ✅ 병해 표본 보강으로 **튼튼히 측정** — 가장 중요한 안전 지표 |
| **5-status 혼동표** | 어느 status로 분류·어디서 헷갈리나 | △ **표본이 확보된 status(병해 등)에 한해** 측정 |

- 1차의 본질 가치는 **이진 FN(진짜 아픈 식물을 놓치지 않는가)**이며, 이는 병해 표본을 늘리면 종류와 무관하게 보강된다.
- 5-status 혼동표는 **표본이 있는 status부터** 의미를 갖는다(현재: 건강·병해, R2 라벨 결과에 따라 건조 등).
- **명시적 스코프 밖**:
  - 과습·건조·영양 부족 status의 상호 혼동(예: "과습을 건조로 오진하는가") — **해당 status 표본이 없어 측정 불가**. `true_status` 스키마는 미리 만들어 두되(그릇), 표본이 확보되는 만큼 **점진적으로** 측정 범위를 넓힌다.
  - 9종을 넘어선 넓은 종 일반화 신뢰 — 더 많은 종 표본이 필요한 별개 후속 축.

> 정직 원칙: 표본 없는 status는 혼동표에서 **빈칸으로 두고 "미측정"으로 표기**한다. 비어 있는 칸을 추정·보간하지 않는다.

## 3. 평가셋 대표성·측정 범위 — 명시와 한계

### 종 대표성

- **대표하는 것**: 식집사 입문 단골 인기종 핵심(몬스테라·스킨답서스·산세베리아·스파티필름·고무나무) 포함 — "흔한 입문 관엽"은 잘 대표.
- **대표 못 하는 것(미측정)**: 금전수(ZZ)·야자류·필로덴드론·싱고니움·페페로미아·호야·칼라데아·고사리류 누락. **야자류(깃털잎)·고사리류(섬세잎) 형태 전무.** Dracaena 속 2종 편중.
- 원칙: 미커버 부류 성능은 9종 점수로 **외삽하지 않는다**.

### status 측정 범위 (v2 신설)

- **측정 가능(표본 있음)**: 건강, 병해 의심, (R2 라벨 결과에 따라) 행운목 5장의 status — 잎끝 마름 계열이면 건조·영양·ambiguous 중 하나.
- **미측정(표본 없음/빈약)**: 과습, 영양 부족(독립 표본), 그리고 위에서 채워지지 않는 나머지 status.
- 평가셋은 당분간 **건강·병해 중심으로 치우침**을 인정하고, 리포트에 "측정된 status = X / 미측정 = Y"를 명시한다.

## 4. 핵심 결정 사항

### 결정 1 — 병해 표본 확보: 공개 데이터셋 (v2 전환)

- **Wikimedia 폐기** — 실측 수율 0(백과사전용 저장소라 일러스트·도감 스캔 위주). `scripts/collect_wikimedia.py`는 남기되 사용 안 함.
- **본인/지인 직접 촬영은 접음** (자연 발생 의존·한계).
- **iNaturalist 증상/종 확장은 접음** (자연 관찰 = 건강 편향, 병해 수율 낮음).
- **신규 1순위: 공개 실내 관엽 병해 데이터셋**
  - **Money Plant 데이터셋** (Mendeley `rzjww3vdxt`, CC BY 4.0) — 스킨답서스 병해. **bacterial wilt → `true_status="병해 의심"`** 편입 후보. 스마트폰 in-the-wild라 PlantVillage보다 도메인 적합. 단 잎 위주 크롭·224 리사이즈본이라 **편입 시 사람 검수** 필요. `original`만 사용(augmented 제외).
  - **Indoor Plant Disease Detection (Kaggle, 다종 실내+다육)** — 종 구성·라이선스·이미지 형식을 **사용자 확인 후** 편입 여부 결정.

### 결정 2 — 종 다양성: 느슨한 확장

- 병해 수집 중 누락된 흔한 종(금전수·필로덴드론 등) 병해가 걸리면 자연스럽게 추가. **강제 종 목표 없음** — 병해 표본 확보 우선.
- 최종 커버 종 목록을 §3에 갱신.

### 결정 3 — PlantVillage: 보조 sanity check (게이트 제외)

- `run_eval.py --aux`로 메인과 **완전 분리** 측정. "명백한 잎 병반에도 '건강'이라 하지 않는가"의 순수 병징 인식 점검.
- **점수는 메인 게이트 미포함** — 참고용. 도메인 미스매치(작물·연구실) 한계 명시. 현 5클래스 유지, 종 추가 안 함.

### 결정 4 — status 정답 라벨: `true_status` + `ambiguous`

- ground_truth에 `true_status` 5종 enum 신규 추가 (백엔드 `ALLOWED_STRUCT_STATUS`와 동일).
- 잎만으로 5종 판정이 곤란하면 `ambiguous` → **평가 제외**(분모 투명 표기).
- `is_healthy ↔ true_status` 방향 정합성 검증 추가 (ambiguous 면제).
- **Money Plant manganese toxicity는 편입 보류** — 영양 "과잉(독성)"이라 현 5-status에 매핑 칸 없음. 시각상 "영양 부족"과 유사하나 원인이 정반대라 거짓 라벨 위험. 보관만(§9 백로그 "영양 과잉 status"와 연결).

## 5. 라벨 스키마 변경

### `test_data/labeling_vocab.py` 추가

```python
STATUS_VOCAB: list[str] = ["건강", "과습", "건조", "병해 의심", "영양 부족"]
STATUS_AMBIGUOUS: str = "ambiguous"  # 잎만으로 5종 판정 곤란 → 평가 제외
```

### `labels.json` ground_truth — `true_status` 한 줄 추가

```json
"ground_truth": {
  "plant_name_korean": "몬스테라",
  "is_healthy": true,
  "true_status": "건강",
  "symptoms": [],
  "diagnosis": "정상 상태"
}
```

### `validate_label` 갱신

- `true_status` 필수 필드 추가. enum 검증: `STATUS_VOCAB` ∪ `{ambiguous}` 외 ValueError.
- 정합성: `true_status=="건강"`이면 `is_healthy=true`, 비건강 4종이면 `is_healthy=false` (ambiguous 면제).
- 기존 허용 유지: `is_healthy=true` + `symptoms=[...]`(경증) 조합 계속 허용.

> 상태: **R1에서 구현 완료**(스키마·마이그레이션 도구). 기존 33장은 건강 28 자동 + 비건강 5 `TODO` → R2에서 사람이 채움.

## 6. run_eval 확장

- 기존 측정(식물명·is_healthy 이진·status 분포·JSON·latency)은 **불변**.
- 신규: **status 혼동표** — `true_status` 정답 대비 예측 status. `ambiguous`는 분모 제외 + 제외 건수 별도 카운트.
  - **표본 없는 status 칸은 빈칸·"미측정" 표기.** 보간·추정 금지.
  - 리포트에 "측정된 status / 미측정 status"를 명시.
- 신규: `--aux` 옵션 — PlantVillage 50장 별도 측정, 결과 JSON 분리, 메인 게이트 미반영.
- `eval/baseline_current_postguard_avg.json`(L0 앵커)·`eval/baseline.json`(구스키마)은 **무변경**.

## 9. 백로그 (본 트랙 밖 / 후속)

- **영양 과잉 status 신설 + 대화 기반 감별** — 영양 과잉(비료 과다)은 흔한 실수이나 영양 부족과 시각 구분이 어려움(이미지 단독 한계). **2차 진단(맥락 수집)** 으로 과잉/부족을 가르는 설계. 이 status가 생기면 Money Plant manganese toxicity 사진이 표본으로 활용 가능.
- 과습·건조·영양 부족 **독립 표본 확보** — 5-status 혼동표 완성을 위한 별도 데이터 작업(공개 데이터셋 희소 → 수동 수집 등 검토).
- 넓은 종 일반화 — 야자류·고사리류·다육 등 미커버 부류 표본 추가 시 측정 범위 확장.
