    - 『하우스플랜트 마스터 가이드』 같은 책 1~2권 OCR/구조화
- 사진만 보고 단순 답변 외에 + (챗봇)간단한 대화 후 → 정보 통합 → 진단 내리기
    
    ```markdown
    [기본 흐름]
    1. 사진 업로드 → 즉시 1차 진단 (Gemini + RAG)
    2. 결과 화면에 "더 정확한 진단을 위해" 3~5개 객관식 질문 표시
       - "마지막 물 준 게 언제인가요?" (1일 이내 / 3일 이내 / 일주일 / 그 이상 / 모름)
       - "주로 어디에 두시나요?" (남향창 / 북향창 / 거실 안쪽 / 화장실 / 모름)
       - "최근 변화가 있었나요?" (분갈이 / 자리 옮김 / 영양제 / 없음)
    3. 답하면 2차 진단 업데이트 (이전 진단 → 보정된 진단)
    ```
    
    왜 1차 진단 후 객관식을 선택했나?
    
    - 사용자가 대답 안해도 1차 진단은 이미 있음 (이탈해도 가치 전달됨)
    - 너무 긴 대화로 인해 진단 결과 흐리게 만드는 것을 막기 위한 목적도 있음
    - 주관식이 아니라서 답하는 부담이 적음
    - (구조화된 입력) LLM이 처리하기도 쉬움
