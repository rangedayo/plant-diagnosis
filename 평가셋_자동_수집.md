#### **시도한 것**

3개 출처로 자동 수집 인프라 구축:

- PlantVillage (Mendeley 원본, CC0): 보조 평가셋 50장 추출
- iNaturalist API: 메인 평가셋용 종별 사진 수집
- Wikimedia Commons API: 메인 평가셋 보조

#### **발견한 실패와 한계**

**1. Wikimedia — 텍스트 매칭의 한계**

- API가 파일명·설명·카테고리 텍스트 매칭이라 콘텐츠 자체를 검증 못 함
- "leaf scorch" 검색 시 USDA 역사 문서, 19세기 식물 카탈로그 표지 등 혼입
- 12장 후보 풀에서 통과율 ~17%

**2. iNaturalist — 해상도 정책 미달 (해결됨)**

- 기본 medium URL이 ~500px short side, 평가셋 정책 400px와 거의 비슷해 압축·트리밍으로 자주 미달
- 초기 통과율 1/22 (4.5%)
- 대응: `medium → large → original` 순차 fallback 구조 추가 → 통과율 5/5

**3. iNaturalist — 도메인 부적합 (근본 한계)**

- "관찰 기록 기반이라 품질 좋다"는 가정이 **'식물 사진' 기준에서만 맞고 '실내 화분 사진' 기준에서는 틀림**
- 실제 수집 결과 대부분 야생 자생지·야외 군락 사진
- 본인 서비스 타겟(식집사 = 실내 화분)과 도메인 불일치
- 6장 중 라벨링 가능한 사진 1~2장 수준

#### **종합 진단**

세 자동 수집 출처 모두 "실내 화분 사진"이라는 서비스 타겟에 완벽히 맞지 않음:

| 출처 | 식물 사진 | 실내 화분 |
| --- | --- | --- |
| PlantVillage | ✓ | ✗ (잎 클로즈업만) |
| iNaturalist | ✓ | ✗ (야외 위주) |
| Wikimedia | △ (노이즈) | 일부 |

#### **향후 계획**

**메인 평가셋(25장) 전략 변경**: 자동 수집 의존 → 수동 수집 중심

- 본인 식물 사진 15~18장 (다양한 각도·증상)
- Unsplash/Pexels에서 실내 식물 5~7장 보충
- 지인 식집사 1명에게 부탁해 5장 내외
- 자동 수집분(iNaturalist·Wikimedia)은 라벨링 가능한 것만 1~3장 활용

**보조 평가셋(50장)**: PlantVillage로 이미 완성, 그대로 유지

#### **면접용 한 줄 요약**

"자동 수집 인프라를 PlantVillage·iNaturalist·Wikimedia로 구축했으나 실제 수집해보니 세 출처 모두 식집사 타겟(실내 화분)에는 한계가 있었다. 시각 인식 평가용 보조셋은 PlantVillage로 유지하되, 메인 평가셋은 본인·지인 사진 중심으로 전환했다. 자동 수집은 한계를 알면서도 인프라 자체가 향후 도메인 확장 시 재사용 가능하다는 점에서 유지했다."
