## 평가셋 통합 작업

self 사진 12장과 iNaturalist에서 가져온 21장을 한 폴더로 합쳐서, 사진과 라벨 정보가 잘 맞는지 9가지 항목으로 검증해 통과시켰다. 그 과정에서 데이터 4가지 불일치를 발견하고 고침.

- self 12장도 iNat 21장처럼 main_eval 폴더로 복사. 원본은 self_captured에 그대로 둠.
- selft_captured 폴더에는 14장 있다. main_eval로 복사할 때 라벨에 있는 12장만 옮김. 빠진 2장은 self_captured 원본 폴더에 그대로 보관.

### 발견한 문제들과 대처

***1. 라벨 파일 두 개의 스키마(필드 구조)가 살짝 달랐음***

iNat 21장 라벨에는 `collection_method`라는 필드가 있는데(어떻게 큐레이션했는지 기록용) self 12장 라벨에는 그 필드가 없었음. 두 파일 합치면 어떤 항목은 그 필드가 있고 어떤 항목은 없는 들쭉날쭉한 상태가 됨.

→ self 12장에도 `"collection_method": "self_captured"` 필드를 추가해서 스키마 통일. 합친 후 33장 전부 같은 필드 구조 갖도록 만듦.

**교훈**: 데이터를 합칠 때는 각 항목의 키 목록이 같은지 미리 확인해야 함. 추적성 정보(이 데이터 어디서 왔는지, 어떻게 수집했는지)는 처음부터 동일한 형식으로 기록하는 게 나음.

***2. site 필드 표기가 일관되지 않았음***

source.site 값이 self 쪽은 `"self"` (소문자), iNat 쪽은 `"iNaturalist"` (카멜케이스)로 표기 통일이 안 됨. 자동 검증 코드 짤 때 `site == "inaturalist"` 같은 비교에서 헷갈릴 수 있는 상태.

데이터의 식별자(identifier) 필드와 사용자에게 보여줄 표시용(display) 필드는 다름. 식별자는 컨벤션 따르고, 공식 표기는 별도 자료에서 사용.

→ 식별자 역할 필드는 소문자+언더스코어 표준에 맞춰 `"iNaturalist"` → `"inaturalist"`로 통일. labels.json과 labels_inat_draft.json 두 파일 다 변경. 변경 전 백업(.bak) 만들어두고 .gitignore에 추가.

### 검증 방법

라벨 파일 한 번 만들고 끝낸 게 아니라, 9가지 항목으로 자동 검증했다. → 전부 통과

1. **기존 검증 함수 통과** — labeling_vocab.py의 validate_dataset 호출 (어휘 검사)
2. **사진 파일 실재 확인** — labels.json에 적힌 33개 경로의 사진이 진짜 폴더에 있는지
3. **사진 이름과 image_id 일치** — `self_dracaena_001.jpg` 사진이 `self_dracaena_001` 라벨과 매칭되는지
4. **폴더에 남는 파일 없음** — 라벨에 없는데 폴더에만 있는 파일 있는지 (역방향 검사)
5. **site 값 분포** — self 12개 + inaturalist 21개로 정확히 나뉘는지
6. **라이선스 값** — 모두 허용 라이선스 목록 안에 있는지
7. **collection_method 분포** — self_captured 12개 + manual_curation 21개
8. **URL 규칙** — self는 null, iNat는 inaturalist.org 주소로 시작하는지
9. **사진가 정보 있음** — photographer 필드가 빈 항목 없는지
